# LeetCode 15. 3Sum

Here is a clear breakdown of the LeetCode "15. 3Sum" problem, along with an efficient solution in Golang.

## The Problem

You are given an array of integers called `nums`. You need to find all unique groups of three numbers (triplets) in the array that add up to exactly zero.

**Important Rules:**

* The three numbers must be at different positions in the array.
* The result must not contain duplicate triplets (e.g., `[-1, 0, 1]` is the same as `[0, 1, -1]`).

## The Strategy: Sort and Two Pointers

A brute-force approach (using three nested loops) would take too much time. The best way to solve this is to **sort the array first** and then use the **Two-Pointer technique**.

Here is the step-by-step logic:

1. **Sort the Array:** Sorting helps us easily avoid duplicates and lets us use pointers logically.
2. **Loop Through the Array:** We use a `for` loop to pick the first number of our triplet, which we will call `nums[i]`.
* **Stop Early:** Because the array is sorted, if `nums[i]` is greater than 0, we can stop entirely. Three positive numbers can never add up to 0.
* **Skip Duplicates:** If `nums[i]` is the same as the previous number `nums[i-1]`, we skip it to prevent duplicate triplets in our final answer.

3. **Use Two Pointers:** For the remaining two numbers, we set a `left` pointer just after `i` (`i + 1`) and a `right` pointer at the very end of the array.
4. **Check the Sum:** * If `nums[i] + nums[left] + nums[right] == 0`: We found a valid triplet! Save it. Then, move both pointers inward, making sure to skip over any duplicate numbers so we don't save the same triplet again.
* If the sum is **less than 0**: The total is too small. We move the `left` pointer to the right to get a larger number.
* If the sum is **greater than 0**: The total is too big. We move the `right` pointer to the left to get a smaller number.

## Golang Solution

```go
import "sort"

func threeSum(nums []int) [][]int {
    // 1. Sort the array
    sort.Ints(nums)
    var result [][]int

    // 2. Loop through the array
    for i := 0; i < len(nums)-2; i++ {
        // Stop early if the smallest number is positive
        if nums[i] > 0 {
            break
        }
        
        // Skip duplicate values for the first number
        if i > 0 && nums[i] == nums[i-1] {
            continue
        }

        // 3. Set up two pointers
        left := i + 1
        right := len(nums) - 1

        // 4. Check the sums
        for left < right {
            sum := nums[i] + nums[left] + nums[right]

            if sum == 0 {
                // Found a valid triplet
                result = append(result, []int{nums[i], nums[left], nums[right]})

                // Skip duplicates for the left pointer
                for left < right && nums[left] == nums[left+1] {
                    left++
                }
                // Skip duplicates for the right pointer
                for left < right && nums[right] == nums[right-1] {
                    right--
                }

                // Move both pointers inward to look for the next combination
                left++
                right--

            } else if sum < 0 {
                // We need a larger sum, move left pointer right
                left++
            } else {
                // We need a smaller sum, move right pointer left
                right--
            }
        }
    }
    
    return result
}

```

## Complexity Analysis

* **Time Complexity:** **O(n²)** Sorting the array takes O(n log n). Then, we have an outer loop running `n` times, and inside that, the two pointers scan the remaining array, which takes O(n) time. Together, the nested loops take O(n²), which becomes our overall time complexity.
* **Space Complexity:** **O(1) or O(n)** Depending on the sorting algorithm used by Golang's `sort.Ints()`, it might use a small amount of extra memory. Excluding the memory used to store the output `result`, the extra space we use for pointers is O(1).
